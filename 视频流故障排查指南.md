# 视频流故障排查指南

## 📋 快速诊断流程

### 第一步：使用 App 内置诊断工具

1. 打开 App 的"视频预览"界面
2. 点击右上角的 **⚙️ 设置/诊断** 按钮
3. 依次运行以下测试：

#### ✅ 测试 1：视频流服务
- 点击"开始测试"按钮
- 检查输出中是否有：
  - `✓ MJPEG 服务运行中` - 服务进程存在
  - `✓ 端口 5002 正在监听` - 网络端口已开放
  - `✓ HTTP 服务响应正常` - 可以访问健康检查接口
  - `✓ 视频帧获取成功` - 能成功获取画面

**常见问题及解决：**

❌ **MJPEG 服务未运行**
```
原因：Python 脚本未启动或已崩溃
解决：点击"重新部署脚本"，然后返回重新开启视频流
```

❌ **端口 5002 未监听**
```
原因：服务启动失败，可能是虚拟环境问题
解决：通过 SSH 手动检查：
  ssh comma@<C3_IP>
  cd /data/openpilot
  . /usr/local/venv/bin/activate
  python3 /data/spapp/spyl/mjpeg_stream.py --camera road --port 5002 --host 0.0.0.0
```

❌ **HTTP 服务无响应**
```
原因：网络连接问题或防火墙阻止
解决：
1. 确认手机和 C3 在同一 WiFi 网络
2. 尝试 ping C3 IP：ping <C3_IP>
3. 尝试用浏览器访问：http://<C3_IP>:5002/health
```

❌ **视频帧获取失败**
```
原因：车辆未启动，stream_encoderd 未产生画面
解决：
1. 确保车辆已启动（ACC ON）
2. 确保 openpilot 正在运行
3. 检查参数是否开启：
   cat /data/params/d/WebrtcStreamEnabled  # 应返回 1
```

#### ✅ 测试 2：HUD 数据服务
- 点击"开始测试"按钮
- 检查是否能获取车速、档位等数据

**常见问题：**
- HUD 服务独立于视频流，失败不影响画面显示
- 如果失败，只是看不到 HUD 叠加信息

## 🔧 手动排查步骤

如果 App 诊断无法解决问题，按以下步骤手动排查：

### 步骤 1：SSH 连接到 C3

```bash
ssh comma@<C3_IP>
# 默认密码通常是空或由用户设置
```

### 步骤 2：检查 Python 虚拟环境

```bash
# 激活虚拟环境
. /usr/local/venv/bin/activate

# 确认 Python 版本
python3 --version  # 应该是 3.11+

# 确认 cereal 库可用
python3 -c "import cereal; print('OK')"
```

### 步骤 3：检查脚本是否存在

```bash
ls -lh /data/spapp/spyl/mjpeg_stream.py
ls -lh /data/spapp/spyl/hud_data_server.py

# 如果不存在，从 App 点击"重新部署"
# 或手动从 App assets 复制
```

### 步骤 4：手动启动视频流服务

```bash
# 停止旧进程
pkill -f mjpeg_stream

# 启动服务（前台运行，方便看日志）
cd /data/openpilot
. /usr/local/venv/bin/activate
export PYTHONPATH=/data/openpilot
python3 /data/spapp/spyl/mjpeg_stream.py --camera road --port 5002 --host 0.0.0.0
```

**观察输出：**
- ✅ `[mjpeg] server started on port 5002` - 服务启动成功
- ✅ `[mjpeg] frame delivered: 640x480` - 正在传输画面
- ❌ `ModuleNotFoundError: No module named 'cereal'` - 虚拟环境问题
- ❌ `Connection refused` - 无法连接 stream_encoderd

### 步骤 5：测试 HTTP 接口

**从 C3 本地测试：**
```bash
# 健康检查
curl http://localhost:5002/health
# 应返回：{"ok": true, "camera": "road"}

# 获取一帧画面（保存为文件）
curl http://localhost:5002/frame > /tmp/test_frame.jpg
ls -lh /tmp/test_frame.jpg
# 应该是 5-15KB 的 JPEG 文件
```

**从手机测试（使用浏览器或 curl）：**
```bash
# 在手机上打开浏览器，访问：
http://<C3_IP>:5002/health

# 如果成功会看到 JSON 响应
# 如果失败，检查网络连接
```

### 步骤 6：检查网络和防火墙

```bash
# 检查端口监听状态
netstat -tuln | grep 5002
# 或
ss -tuln | grep 5002

# 应该看到：
# tcp   0   0   0.0.0.0:5002   0.0.0.0:*   LISTEN

# 如果只看到 127.0.0.1:5002，说明只监听本地
# 需要确保启动时加了 --host 0.0.0.0
```

### 步骤 7：查看日志

```bash
# MJPEG 服务日志
tail -f /data/spapp/spyl/log/mjpeg_stream.log

# HUD 服务日志
tail -f /data/spapp/spyl/log/hud_data_server.log

# openpilot 主日志
tail -f /data/openpilot/log/swaglog

# 系统日志
dmesg | tail -n 50
```

## 🚨 常见错误及解决方案

### 错误 1：连接拒绝 (Connection refused)

**症状：**
```
❌ HTTP 服务无响应
java.net.ConnectException: Connection refused
```

**原因：**
- MJPEG 服务未启动
- 服务监听在 127.0.0.1 而不是 0.0.0.0

**解决：**
1. 确认服务启动命令包含 `--host 0.0.0.0`
2. 检查防火墙是否阻止端口 5002
3. 重新部署脚本（App 内"重新部署"按钮）

---

### 错误 2：超时 (Timeout)

**症状：**
```
⚠ Timeout fetching frame from http://<IP>:5002/frame
```

**原因：**
- 网络延迟过高
- 服务响应慢
- 车辆未启动，无画面输出

**解决：**
1. 确保车辆已启动（ACC ON）
2. 检查 WiFi 信号强度
3. 尝试降低画面质量（修改 mjpeg_stream.py 的 JPEG_QUALITY）

---

### 错误 3：空响应 (Empty response)

**症状：**
```
⚠ Empty response from http://<IP>:5002/frame
```

**原因：**
- stream_encoderd 未产生帧
- openpilot 未运行

**解决：**
1. 确认 openpilot 在运行：`ps aux | grep openpilot`
2. 检查参数：`cat /data/params/d/WebrtcStreamEnabled`
3. 重启 openpilot 或重启 C3 设备

---

### 错误 4：ModuleNotFoundError: cereal

**症状：**
```python
ModuleNotFoundError: No module named 'cereal'
```

**原因：**
- 未激活 Python 虚拟环境
- PYTHONPATH 未设置

**解决：**
```bash
cd /data/openpilot
. /usr/local/venv/bin/activate
export PYTHONPATH=/data/openpilot
python3 /data/spapp/spyl/mjpeg_stream.py --camera road --port 5002 --host 0.0.0.0
```

---

### 错误 5：画面卡顿或延迟高

**症状：**
- FPS 显示 < 5
- 画面更新很慢

**原因：**
- C3 CPU 负载高
- 网络带宽不足
- 画面分辨率/质量过高

**解决：**
修改 `/data/spapp/spyl/mjpeg_stream.py`：
```python
# 降低分辨率
TARGET_WIDTH = 480  # 原来是 640

# 降低JPEG质量
JPEG_QUALITY = 40   # 原来是 50

# 增加帧间隔（降低帧率）
FRAME_INTERVAL = 0.15  # 原来是 0.1 (6fps vs 10fps)
```

---

## 📊 性能优化建议

### C3 设备性能不足时：

1. **降低分辨率**：`TARGET_WIDTH = 480`
2. **降低质量**：`JPEG_QUALITY = 40`
3. **降低帧率**：`FRAME_INTERVAL = 0.15`
4. **关闭 HUD 叠加**：点击视频画面左上角的 HUD 开关

### 网络延迟高时：

1. **使用 C3 热点**：让手机直连 C3 的 WiFi（延迟最低）
2. **避免 WiFi 拥挤**：切换到 5GHz 频段
3. **增加缓冲**：修改 Android 端的 `readTimeout`

### App 卡顿时：

1. **关闭其他后台应用**
2. **降低 Android 端轮询频率**：修改 VideoScreen.kt 的 `delay(100)` 为 `delay(150)`

## 🛠 高级调试

### 使用 tcpdump 抓包

```bash
# 在 C3 上抓包，查看 HTTP 流量
tcpdump -i wlan0 port 5002 -w /tmp/mjpeg.pcap

# 传输到电脑分析
scp comma@<C3_IP>:/tmp/mjpeg.pcap .
wireshark mjpeg.pcap
```

### 直接访问 stream_encoderd

```python
# Python 脚本测试 H264 流
cd /data/openpilot
. /usr/local/venv/bin/activate
python3 << 'EOF'
from cereal import messaging
import time

sock = messaging.sub_sock("roadEncodeData")
print("Waiting for frames...")

for _ in range(10):
    evt = messaging.recv_one(sock)
    if evt:
        print(f"Frame: {len(evt.roadEncodeData.data)} bytes, "
              f"idx={evt.roadEncodeData.idx}")
    time.sleep(0.5)
EOF
```

### 查看系统资源占用

```bash
# CPU 占用
top -n 1 | grep -E "python3|openpilot"

# 内存占用
free -h

# 磁盘空间
df -h /data

# 网络统计
ifconfig wlan0
netstat -s | grep -i error
```

## 📞 仍然无法解决？

### 收集诊断信息：

1. 运行 App 内的所有诊断测试，截图保存
2. 查看 C3 日志：`tail -n 100 /data/spapp/spyl/log/mjpeg_stream.log`
3. 检查网络连接：`ping <C3_IP>` 和 `curl http://<C3_IP>:5002/health`
4. 记录错误信息：完整的错误消息和堆栈跟踪

### 提交问题时包含：

- C3 设备型号和 openpilot 版本
- Android 设备型号和系统版本
- App 版本
- 完整的错误日志
- 诊断测试截图
- 网络环境描述（WiFi 还是热点，信号强度等）
